## 修复内容

- Prior state fixture 的 cooldown_until 时间锚定在 2026-06-04

### 修复方案

1. **v112i Runner** (`run_market_radar_v112i_dedupe_cooldown_gate.py`)

   - Result JSON 新增字段：
     - `deterministic_clock: true`
     - `evaluated_at: "2026-06-04T22:30:00+08:00"`
     - `time_dependent_test_risk: false`
   - 移除未使用的 `_now_cn` 导入

2. **Gate Module** (`market_radar_dedupe_cooldown_gate_v112i.py`)
   - 无需修改 — 所有核心函数已支持 `evaluated_at` 参数
   - `check_cooldown()` / `evaluate_signal_gate()` / `evaluate_all_signal_gates()` 均接受显式 `evaluated_at`

3. **v112k / v112l**
   - v112k 通过 `parse_generated_at()` 从 prior state 提取时间 — 已确定性
   - v112l 仅做 key validation，不涉及 gate evaluation — 无时间依赖
