# Stage 2 Exit — Foundations and Repository Audit

**Status:** Accepted  
**Merged evidence:** PR #26  
**Merge commit:** `73eca74b8ee2be472645eda997179b54c0b9099d`

## Accepted foundation route

- isolated Python 3.12 environment;
- Pydantic domain contracts;
- Pydantic AI as a thin structured semantic gateway;
- SQLAlchemy and Alembic for application persistence and migrations;
- explicit append-only revisions plus current projections;
- table-driven thesis lifecycle validation;
- minimal local SQLAlchemy durable-review runtime for the first internal version;
- OpenTelemetry traces and metrics;
- relational provenance inspired by W3C PROV;
- deterministic services around two bounded semantic passes.

## Deferred route

DBOS remains conditional because its current runtime requires Postgres. It is not selected for the first internal version. Reopen only if the minimal runtime fails recovery, scheduling or concurrency requirements that DBOS can demonstrably satisfy at acceptable operating cost.

## Repository judgment

Draft PR #16 remains unmerged. It is a component quarry and failure record, not the product base.

Selected existing assets may be adapted:

- read-only acquisition providers;
- source status and provenance concepts;
- exact duplicate and event-key handling;
- artifact hashes and point-in-time intent;
- operator doctor, status, manifests and finite stop controls;
- useful fixture and falsification tests.

The first active path excludes:

- automatic activation from live origin;
- keyword causal mechanisms;
- universal placeholder world state;
- eight unvalidated strategy evaluators;
- arithmetic confidence;
- strategy-count arbitration;
- one-shot packet as the product object;
- batch execution presented as longitudinal Shadow.

- real Alembic upgrade and downgrade behavior;

- zero product behavior, paid model calls, Postgres services or background daemons.
