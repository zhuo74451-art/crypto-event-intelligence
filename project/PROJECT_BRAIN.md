# Project Brain

## Identity

**Crypto Market Cognition & Signal OS** is an unattended internal operator that maintains a bounded portfolio of evidence-backed crypto-market theses.

It is not a copy-trading, order-advice, wallet, public-recommendation or external SaaS product.

## Owner role

The owner remains outside routine judgment. Scope, protected access, material cost, public operation, irreversible effects, stop and recovery remain governance decisions.

## Core business object

The product maintains persistent theses with evidence, event state, mechanism, exposure, horizon, priced-in state, counterevidence, attention, follow-up, invalidation and revision history.

## Smallest accepted semantic architecture

Stage 2 rejects the interpretation that six responsibility roles require six model agents.

The first architecture has:

- deterministic Evidence Gate;
- deterministic Event and Candidate Builder;
- one Thesis Synthesis pass;
- one Risk Challenge pass;
- deterministic Arbitration;
- deterministic Lifecycle and Resource service;
- a durable runtime selected through evidence.

No free-form agent conversation, autonomous tool loop or model-controlled state change belongs in the first slice.

## Existing repository judgment

Useful assets on `main` and Draft PR #16 include:

- bounded read-only providers;
- source status and provenance concepts;
- exact duplicate and event-key handling;
- artifact hashes and point-in-time intent;
- operator doctor, status, manifests and finite stop controls;

The future active path does not reuse:

- title-keyword causal mechanisms;
- universal placeholder world-state objects;
- eight unvalidated strategy heuristics;
- arithmetic confidence;
- strategy-count arbitration;
- one-shot packet as the product object;
- batch execution labelled as longitudinal Shadow.

PR #16 remains Draft and unmerged. It is a component quarry and a record of earlier assumptions.

## Foundation state

Accepted:

- Pydantic domain contracts;
- SQLAlchemy and Alembic application persistence;
- append-only revisions and current projections;
- two semantic contracts;
- OpenTelemetry traces and metrics;
- relational provenance inspired by W3C PROV.

Pending evidence:

- Pydantic AI compatibility and bounded repair;
- state-machine library versus table validation;
- DBOS versus a minimal local durable-review ledger.

Current official package metadata requires Python 3.10 or later for the principal candidate libraries. The known Mac system Python 3.9.6 is not modified; Stage 2 uses an isolated supported runtime or reports a blocker.

DBOS is no longer accepted on an assumed SQLite basis. Its current Postgres-backed operating boundary must earn approval through evidence and a later service decision.

## Current business node

Execute the bounded foundation compatibility spike without product behavior, paid model calls, Postgres installation, daemon operation or PR #16 changes.

The result must decide or narrow:

- supported Python and dependency set;
- semantic gateway viability;
- persistence and migration viability;
- transition-validator route;
- durable-runtime route;
- observability route;
- first-slice dependency graph.
