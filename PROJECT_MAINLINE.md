# Crypto Market Cognition & Signal OS — Mainline Contract

The canonical state is `project/CANONICAL_STATE.yaml`.

## Responsibility

Build an unattended internal market-cognition operator that discovers, verifies, interprets and maintains a bounded portfolio of crypto-market theses.

Its first outcomes are autonomous research prioritization and autonomous watch-portfolio maintenance. It is not a trade-instruction or public-advice product.

## Core path

```text
Evidence
  -> Event
  -> Interpretive Claim
  -> Thesis
  -> Exposure
  -> Attention State
  -> Lifecycle Revision
```

## Discovery

Discovery is broad; active admission is strict. Any related object may be briefly screened. Active admission requires valid identity, real change or new evidence, a defensible mechanism, genuine exposure, evidence permission, a future verification path, an exit condition, acceptable risk, positive attention value and non-duplication.

## Judgment authority

The full contract is [`project/AUTONOMOUS_JUDGMENT_CONTRACT.md`](project/AUTONOMOUS_JUDGMENT_CONTRACT.md).

- L0: deterministic rules are final.
- L1: bounded semantic interpretation.
- L2: allowlisted internal arbitration and lifecycle actions.
- L3: owner governance exceptions.

Normal-path judgments J01 through J22 each have one owner, allowed outputs and a conservative fallback. Routine uncertainty stays inside the system.

## Stable rules

1. Accepted responsibility and planning changes are synchronized to `main` in the same discussion cycle.
2. QuickFlash remains a separate broad-recall acquisition provider.
3. Evidence, events, interpretations and theses remain separate.
4. Point-in-time integrity and future-data blocking are mandatory.
5. Semantic roles cannot override deterministic prohibitions.
6. Clear reversible low-risk defaults are applied directly and reported.
7. Longitudinal shadow evidence is required before autonomous judgment is trusted.
8. Existing modules are later classified as `RETAIN`, `ADAPT`, `QUARANTINE`, `REMOVE` or `MISSING`.
9. Future work bases must contain the canonical fingerprint.
