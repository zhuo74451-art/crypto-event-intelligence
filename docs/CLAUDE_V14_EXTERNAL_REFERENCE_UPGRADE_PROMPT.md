# 项目目标

我们在做一个 Crypto Event Intelligence 系统：把 Web3 快讯、链上一手监控、CEX/DEX/Hyperliquid/稳定币/ETF/安全事件等信息，转成可验证的事件样本和 Telegram 情报摘要。

目标不是交易系统，不下单，不生成买卖/做多/做空建议。目标是：

1. 让用户快速看到真正值得关注的市场事件。
2. 每条被推送/摘要的情报都能被后续 1h/4h/24h/72h 回填验证。
3. 用历史回测和实时 follow-up 逐步证明哪些来源、事件类型、表达方式有价值。
4. 最终形成“情报采集 -> 过滤 -> 结构化 -> 推送/摘要 -> 结果评估 -> 策略化改进”的闭环。

# 当前痛点

1. 之前 Telegram 消息太乱：分数无解释、字段重复、用户读不懂。
2. 盘中雷达很容易变成“信息堆叠”，不是注意力过滤器。
3. 来源质量不稳定，webhook 混杂，很多内容不是交易相关情报。
4. 事件类型样本仍薄，部分类型 2-8 条，不能支撑统计结论。
5. `other/needs_taxonomy_review` 本质是分类系统没做好。
6. BTC/ETH 基准资产事件会污染 abnormal_vs_btc/eth。
7. 缺 regime filter，当前回测容易把市场环境误认为事件效果。
8. 缺 pre-event price-in 检查的产品化使用。
9. ETF/安全事件/资金流都有“文本看起来重要但归因错、金额错、资产错”的风险。
10. 当前不是基础链路问题，而是数据质量和产品呈现问题。
