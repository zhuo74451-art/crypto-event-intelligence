# Claude 咨询规则

更新：2026-05-28 UTC+8

以后所有发给 Claude 的项目咨询，默认遵守以下规则。

## 输出语言

1. 默认中文。
2. 必须使用英文术语时，第一次出现必须附中文解释。
3. 不允许用英文模板代替中文业务表达。
4. 不允许输出用户看不懂的缩写堆叠。

## 输出结构

每次咨询必须要求 Claude 输出：

1. 真实判断：当前方向哪里对，哪里错。
2. 关键问题：必须逐条列出，不合并成大段观点。
3. 每个问题的解决方案。
4. 具体字段、脚本、表、数据文件建议。
5. 验收标准。
6. 不建议做的事。

## 对我们自己的要求

Claude 回复后，不只做摘要，必须补一份中文执行方案：

- 每个问题对应一个方案。
- 每个方案对应可落地任务。
- 每个任务对应验收标准。
- 如果 Claude 的建议不适合当前阶段，要明确写“不采纳”和原因。

## 禁止

1. 禁止只把 Claude 观点复制到项目文档。
2. 禁止只做宽泛规划，不拆任务。
3. 禁止把英文术语原样堆给用户。
4. 禁止用“AI 分类”“更多数据源”“更好看”掩盖核心问题。

## 当前项目优先级

当前最高优先级不是扩源，而是：

1. TG 情报发布账本。
2. 发布后 1h / 4h / 24h / 72h 自动评价。
3. 事件子类型和假设注册表。
4. price-in 检查。
5. regime 市场状态标签。
6. 给量化人员看的数据包。
